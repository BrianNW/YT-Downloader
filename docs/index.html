Local-first workflow

Download and compress videos on your own machine.

This repository ships a Flask app and Windows desktop launcher for MP4 downloads and compression. GitHub Pages hosts this static project site, while the actual downloader runs locally because it depends on Python, yt-dlp, and FFmpeg.

If you want a live hosted app later, move the backend to a Python host such as Render, Railway, or Azure App Service.

Run it locally

Use the repo for development or install the packaged Windows desktop build.

Web app

Start the Flask server and open the local browser UI.

python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python app.py

Desktop launcher

Run the desktop wrapper that opens the local app automatically.

python desktop.py

Or download the latest packaged executable from GitHub Releases.

What the app does

  • Download the best available MP4 stream for supported links.
  • Compress local video files to a target size, bitrate, or resolution.
  • Track recent job history in a local JSON log.

GitHub Pages publishing

This repository is configured to deploy the contents of the docs folder with GitHub Actions.

Already configured

  • Static site lives in the docs folder.
  • Workflow publishes on pushes to main or master.
  • No Jekyll build step is required.

One-time GitHub setting

  1. Open the repository on GitHub.
  2. Go to Settings, then Pages.
  3. Set Source to GitHub Actions.

Recommended next step
